Project Overview

A production-ready ETL pipeline demonstrating data engineering best practices for defense asset telemetry, events, and incident management. This project showcases the complete data lifecycle from heterogeneous source ingestion through star schema warehousing, automated quality validation, semantic layer design, and machine learning integration.

Built to demonstrate capabilities relevant to defense and aerospace organizations, the pipeline consolidates data from:

  • Relational operations database (SQL-like) for assets, events, and incidents
  • CSV log extracts from external systems
  • JSON-based telemetry feeds from sensors and monitoring systems

Key Features & Capabilities

Star Schema Warehouse

Dimensional modeling with 2 dimension tables (dim_asset, dim_date) and 2 fact tables (fact_events, fact_incidents) optimized for analytical queries and BI tool integration.

Data Quality Framework

Automated validation checks for null rates, referential integrity, value ranges, and enum constraints with comprehensive logging to a quality database for audit trails.

Semantic Layer

Business-friendly views (vw_asset_health, vw_incident_summary, vw_incident_risk) with data catalog, lineage tracking, and pre-calculated KPIs for executive reporting.

ML Pipeline

Random Forest classifier for incident severity prediction with feature engineering, model persistence, and batch inference capabilities for proactive risk assessment.
