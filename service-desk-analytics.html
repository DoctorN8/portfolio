Data Engineering & Analytics

Service Desk Analytics Pipeline

A full-stack data engineering project transforming raw logs into actionable efficiency metrics using DuckDB and Streamlit.

Launch Live Demo View Code

The Challenge

Service organizations often struggle to identify the root cause of inefficiencies. Metrics like "Mean Time to Resolve" (MTTR) are often aggregated, hiding the specific technicians or ticket types causing bottlenecks. The goal of this project was to build a transparent, drill-down capable analytics suite that a VP of Services could use to justify headcount or identify training gaps.

The Solution

I engineered an end-to-end ELT (Extract, Load, Transform) pipeline that ingests raw ticketing data, models it into a Star Schema for high-performance querying, and visualizes it via an interactive web application.

System Architecture

1. Data Generation & Ingestion

Created a robust Python script using Faker and NumPy to generate realistic synthetic data. This includes specific patterns (e.g., a "bottleneck technician" with high re-open rates) to ensure the dashboard tells a discoverable story.

2. Data Modeling (DuckDB)

Designed a Star Schema consisting of fact_tickets, fact_sla_snapshots, and dimension tables. Used DuckDB as a local OLAP warehouse to handle window functions and fast aggregations.

3. Visualization Layer

Built a multi-page Streamlit application using Plotly for interactive charting. The app connects directly to the DuckDB file for low-latency querying.

Key Insights Visualized

  • Executive Pulse: High-level view of MTTR, First Contact Resolution (FCR), and Backlog growth.
  • Efficiency Matrix: A scatter plot comparing Ticket Volume vs. Customer Satisfaction (CSAT) to identify high-performers vs. struggling agents.
  • Backlog Trends: Historical snapshot analysis to visualize how the ticket queue grows over time.
